# Ludonect — Purchase Flow Plan

## Ziel
Dummy-Purchase-Flow von "Kaufen klicken" bis "Spiel starten mit 6-stelligem Code + Subdomain".
Kein echtes Payment. Keine echte E-Mail. Stattdessen: Download-Datei mit allen Infos.

## Architektur
- Kein DB-Server — alles in-memory (wie GameManager)
- Neuer InstanceManager für "gekaufte" Instanzen
- REST-Endpunkte im bestehenden HTTP-Server

## Flow
1. User klickt "Pro/Buy" auf Landingpage
2. Formular: Email, Subdomain-Wunsch, Event-Name, Laufzeit, Frage-Set
3. "Kaufen" → POST /api/purchase
4. Server erstellt Instance, generiert 6-stelligen Code
5. Browser lädt Info-Datei herunter (JSON)
6. User kann über Subdomain oder Code beitreten

## Dateien die geändert/erstellt werden

### Backend
- `server/instanceManager.ts` (NEU) — in-memory instance store
- `server/index.ts` — HTTP-Endpunkte + Subdomain-Routing
- `server/socketHandlers.ts` — 6-stellig Code-Logik

### Frontend
- `src/views/PurchaseView.vue` (NEU) — Kauf-Formular
- `src/router/index.ts` — /purchase Route
- `src/views/LandingPageView.vue` — Pro-Button
- `src/views/LobbyView.vue` — ggf. Subdomain-Erkennung
